Bachelor Thesis

NEAT-TCP: Generation of TCP Congestion Control Through Neuroevolution of Augmenting Topologies   Github

NEAT-TCP is a novel technique to automatically generate congestion control algorithms in a data-driven fashion while optimizing towards a specified global system utility. NEAT-TCP employs an artificial neural network (ANN) in each node and generates a population of ANNs by means of an evolutionary algorithm called NEAT. The ANNs run independently from each other at the communication endpoints and take only features as inputs that are locally available at these nodes. The system utility is defined as a combined maximization of overall throughput and throughput fairness between flows according to Jain’s fairness index. The nodes are deployed in a grid topology in ns-3 simulations, which makes it particularly difficult to maximize the utility due to different interference levels for the data flows. In the conducted experiments, NEAT-TCP achieves 69% more fairness, 66% less mean end-to-end delay and 71% less packet loss in relation to TCP New Reno at the cost of 19% less overall throughput, which meets the multi-criteria objective.

Used techniques

  • ns3
  • C++
  • Python (for visualizations)

Practical Lab

Practical Evaluation of NEAT-TCP   Github

This project is a extention work of my Bachelor's Thesis. As I wasn't able to evaluate NEAT-TCP outside of ns-3 simulations, the target of this project was to evaluate TCP Congestion Control that was created by NEAT-TCP on a real testbed out of 25 raspberry pis.
The contributions of this project constited out of:

  • Porting of NEAT-TCP from ns-3 to Linux as a pluggable kernel module.
  • Planning and building of a testbed.
  • Evaluation of NEAT-TCP and other Congestion Control algorithms on this testbed in comparison to ns-3 results.

Used techniques

  • Raspbian
  • C
  • Phyton (for visualizations)
